| TD-029 | S6-8 | performance | During the TD-028 multi-worker experiment (`uvicorn --workers 4`), `app/db/session.py`'s `pool_size=20, max_overflow=30` (set in TD-026) is applied **per worker process**, not per application. | TD-030 | S7-1 | performance | `_get_converted_amount_usd` (`app/services/transaction_service.py`) is called once per entry inside `create_transaction`'s list comprehension (`converted_amounts = [...]`), but its `currency_code` and `transaction_date` arguments are `payload.currency_code` / `payload.transaction_date` — transaction-level values identical for every entry. For non-`BASE_CURRENCY` (`USD`) transactions, each call issues up to 3 queries (resolve `from_currency`, resolve `to_currency`/USD, look up `ExchangeRate` for the date), so a transaction with N entries issues up to 3×N queries to resolve what is effectively a single conversion rate. (When `currency_code == "USD"`, the function returns early with 0 queries, so only non-USD transactions are affected.) Discovered during S7-1 while reviewing this function's query patterns alongside the TD-024 fix. Fix: resolve `from_currency`/`to_currency`/`exchange_rate` once before the loop, and apply the resulting rate to each entry's amount inside the loop. | Medium | S7-1 |
| TD-031 | S7-1 | data-integrity | `create_user` (`app/services/user_service.py`) checks for an existing user via `select(User).where(User.email == payload.email)` then `result.scalar_one_or_none() is not None` — fetching the full `User` row just to test existence (could be narrowed to `select(User.id)`). More significantly, this is a check-then-insert (TOCTOU) pattern: if two requests with the same email race between this SELECT and the subsequent `INSERT`, the `users.email` UNIQUE constraint raises `IntegrityError`, which is not caught here, so the losing request surfaces as a 500 instead of the intended 409 Conflict. Discovered during S7-1 while reviewing query-fetch patterns alongside the TD-024 fix. Fix candidates: (1) narrow to `select(User.id)`; (2) catch `IntegrityError` on the subsequent flush/commit and translate to 409 as defense against the race. | Low | S7-1 |
